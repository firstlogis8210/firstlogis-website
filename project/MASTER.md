# 4. 작업 원칙

1. 홈페이지 수정은 GitHub를 기준으로 관리한다.
2. Netlify는 배포 및 도메인 확인 용도로만 사용한다.
3. 한 번에 여러 기능을 수정하지 않는다.
4. 하나의 작업은 하나의 커밋으로 관리한다.
5. 수정 전 기존 구조와 모바일 화면을 반드시 확인한다.
6. 기존 디자인과 레이아웃을 임의로 변경하지 않는다.
7. 모바일 환경을 우선 고려한다.
8. SEO(Title, Description, Canonical, Sitemap, Robots)를 훼손하지 않는다.
9. 전화번호와 회사 정보는 추측하지 않고 확정된 정보만 사용한다.
10. 비밀번호, API 키, 인증서 및 개인정보는 GitHub에 저장하지 않는다.

# 5. Git 운영 원칙

- main 브랜치는 항상 배포 가능한 상태를 유지한다.

- 배포 전 변경 파일을 확인한다.

- 클릭 수보다 실제 문의(전화, 카카오톡, 견적접수)를 우선 지표로 판단한다.

# 10. AI 작업 원칙

ChatGPT 또는 Codex는 작업 전에 아래 문서를 반드시 확인한다.

## 작업 순서

1. MASTER.md
2. TODO.md
3. CHANGELOG.md

위 순서를 확인한 후 작업을 시작한다.

## 작업 규칙

- 현재 홈페이지 구조를 먼저 확인한다.
- 기존 기능을 임의로 삭제하지 않는다.
- 전체 디자인을 임의로 변경하지 않는다.
- 기존 프로젝트 문서를 우선 신뢰하고 추측으로 구조를 변경하지 않는다.
- 수정 범위와 영향을 먼저 설명한다.
- SEO와 모바일 화면 영향을 확인한다.
- 작업 후 변경된 파일과 확인 방법을 알려준다.
- 가능하면 기능별 커밋을 권장한다.
- MASTER.md는 프로젝트의 기준 문서이며 AI는 임의로 수정하지 않는다.
- MASTER.md 수정이 필요한 경우 수정 이유를 먼저 제안하고 사용자의 승인 후 수정한다.
